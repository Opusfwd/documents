1、UnderWriteMe SDK​

  • SDK Repo

    • https://dev.azure.com/FWDGODevOps/UWME_LIB

  • UnderWriteMe Engine

    • https://client-repository.underwriteme.co.uk

  • Using For

    • IDN
    • PHP
    • MYS - Agency channel (Banca using local questionnaire)

  • Smart Working Flow

    StepPageDetail
    1Dash BoardRuleBook Register
    2Health QuestionsGenerate UWMe HealthQuestions
    3Review & SignaturGet UWMe Decision
    4Submit ApplicationImport UWMe zip into UWWS

  • Flow Detail

    • RuleBook Register

      1. checking use underWriteMe (MY Banca not)
      2. UnderwriteMeModule.latestDeployment
        • Request: empty
        • Response: 160
      3. Call UWWS deployments
      4. UnderwriteMeModule.registerDeploymentDescriptors
      5. UnderwriteMeModule.listRequiredDefinitions
        • Response: ["/FWD_MY_IFA/IFA_Rulebook_V1b"]
      6. Call UWWS enquiryDefinition
      7. UnderwriteMeModule.registerRule(downloadDest)
        • Request: rulebook uri
        • Response: {"status","success"}

    • Generate UnderWriteMe HealthQuestions

      1. Call OwbService getPreUWSubmission
      2. UnderwriteMeModule.answerQuestions
        • newenquiry
          • store enquiryId
        • render
          • existing enquiry , no need handle
        • enquiry not found （enchancement)
          • hasStoreZipJson
            1. UnderwriteMeModule.enquiryExists
              • not exist then zip the json into Enquiry_xxx.zip
            2. UnderwriteMeModule.importEnquiry
          • clean enquiryId and re-trigger answerQuestions
        • enquiry definition not found
          • Download rule book and re-trigger answerQuestions
      3. UnderwriteMeModule.setPartyIndex
        • Request (partyIndex: init = 0)
      4. UnderwriteMeModule.getEnquiry

    • Render HealthQuestions

      • Render Questions
      • Changes Tab
        1. UnderwriteMeModule.setPartyIndex
        2. UnderwriteMeModule.getEnquiry
      • Answer Question
        • UnderwriteMeModule.postEnquiry
      • Done Button
        • UnderwriteMeModule.getQuestionAnswer
        • saveZipFiles （enchancement)
          • unzipEnquiry and store zip JSON into answerQuestions

    • Get UnderWriteMe Decision

      1. prepareUnderWriteMe （enchancement)
        • hasStoreZipJson
          1. UnderwriteMeModule.enquiryExists
            • not exist then zip the json into Enquiry_xxx.zip
          2. UnderwriteMeModule.importEnquiry
      2. UnderwriteMeModule.getDecision
      3. saveZipFiles （enchancement)
        • unzipEnquiry and store zip JSON into answerQuestions

    • Import UWMe zip into UWWS

      1. prepareUnderWriteMe （enchancement)
        • hasStoreZipJson
          1. UnderwriteMeModule.enquiryExists
            • not exist then zip the json into Enquiry_xxx.zip
      2. Call UWWS Import

  • Integration OWB Path

    • businessData.customers[x].enquiryId

8、Problems that may be encountered​

1、Health Questions display wrong questions, or display the questions should not display​

  1. Check the answerQuestion response base on Render Questions Logic
    • hasQuestions ?
    • isSatisfied ?
    • hasAnswers ?
    • validationErrors ?
  2. Check the indicator whether has integrated
    • If not then confirm with underWriteMe team how to integrate it
  3. Provide online enquiry (generate on UWWS) to underWriteMe team to check

2、Got Error Enquiry Definition not found​

  1. Check Whether the UWWS server is available
  2. Check the Logs whether the ruleBook downloaded and register success
  3. For Currently SDK , there mind got this issue once fail to donwload rulebook , need re-install the app and try again

3、Got "openenquiry" ( Some questions are not answered ) when get decision​

  1. Check whether the reuqest exist enquiryId for each party
  2. Check whether there have enquiry not closed
  3. Use the answerQuestions request to generate on UWWS, check whether owner need answer questions

    • PH and ID not display owner health question

4、Got Null Point Exception when get decision​

  1. Check the request of riskType and benefitSubType
  2. Check the validationErrors of answerQuestion response for Product integration
  3. Run the Android Emulator for that case to check the error detail
  4. Run adb locat to check the error code and lines
  5. Check SDK source code or ask underWriteMe Team to help if still no idea

5、Upload Zip File error​

  1. Check Whether the UWWS server is available
  2. Prepare the Zip File
    • create files with zip Json which store on database and zip it manually
    • using Smart-Tools by run smart underwriteme getZipFiles -q policyNum=xxx
  3. Call UWWS Import to check the response
  4. If response as expected for status 200,201,409 , ask User to retry should be fine

RuleBook Register Enhancement​

Current Register Flow​

  1. UnderwriteMeModule.latestDeployment

    • Request: empty
    • Response: 160

  2. Call UWWS deployments with sinceId = latestDeployment response

  3. UnderwriteMeModule.registerDeploymentDescriptors

  4. UnderwriteMeModule.listRequiredDefinitions

    • Response: ["/FWD_MY_IFA/IFA_Rulebook_V1b"]

  5. Call UWWS enquiryDefinition

  6. UnderwriteMeModule.registerRule(downloadDest)

    • Request: rulebook uri
    • Response: {"status","success"}

Error Point​

  • Once Call UWWS enquiryDefinition DownLoad RuleBook Or UnderwriteMeModule.registerRule(downloadDest) Fail
  • But the UnderwriteMeModule.registerDeploymentDescriptors and UnderwriteMeModule.listRequiredDefinitions already success ,
  • And next time UnderwriteMeModule.latestDeployment already return latest deployId , meaning not more deployment need be download

Expected result​

  • UnderwriteMeModule.latestDeployment should return the lastTime register success deployId , not using the deploymentList latest deployId

Solution​

  • If can not resolve the sdk issue, smart should process to download and register latest version without checking
